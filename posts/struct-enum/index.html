Struct, Enum

2021-01-20
1 minute read
Rust

Notes on Rust in Action, by Tim McNamara

type

Just an aliasing mechanism. For example, the following code will compile and run:

1type MyString = String;
2fn main() {
3	let haha: MyString = String::from("haha");
4	let hehe: String = MyString::from("hehe");
5}

“new type” struct wrapping

If aliasing is not ideal, try this:

1struct MyString(String); // a tuple

Now, the following code will not compile since MyString and String are no longer interchangeable.

1fn main() {
2	let haha: MyString = String::from("haha");
3}
41  |     let haha: MyString = String::from("haha");
5   |               --------   ^^^^^^^^^^^^^^^^^^^^ expected struct `MyString`, found struct `String`

methods

Use impl block with the name of the struct to declare methods.

1impl StructName {
2	// todo
3}

Although structs can be initialized using the struct literal, a more common way is to declare a new() methods, as a helper function, to create a new instant of the struct.

Result

Result is an enum, it consists of two variants: Ok(T) and Err(E).

enum

  1. Works with pattern matching
  2. Can have methods
  3. Can include data within its variants

trait

Similar to an interface, it defines a behavior that struct implemented this trait shall follow. To define a trait, create a block with the trait keyword and the name of the trait. Place method’s name, parameters, and return value only inside the trait block:

1trait Read {
2    fn read(self: &Self, save_to: &mut Vec<u8>) -> Result<usize, String>;
3}

The Self is a pseudo type, playing as a player holder for the type that will implement this trait To implement a trait, create a block with impl [trait name] for [struct name].
